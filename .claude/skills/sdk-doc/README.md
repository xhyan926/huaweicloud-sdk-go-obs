# SDK API 文档编写指南 Skill

这是一个为 SDK 项目提供 API 接口文档编写指导的 Claude Code Skill。

## 功能概述

本技能提供了完整的 SDK API 文档编写流程，包括：

1. **文档结构设计** - 按特性组织文档目录
2. **文档模板** - 提供标准化的文档模板
3. **内容规范** - 详细的格式和内容要求
4. **最佳实践** - 基于经验的文档编写建议
5. **质量检查** - 文档质量验证清单

## 使用方法

### 基本用法

当需要编写 SDK API 文档时，直接调用此 skill：

```
/sdk-doc: 为桶清单功能生成 API 接口文档
```

### 典型场景

1. **为新功能创建文档**

   ```
   /sdk-doc: 为 PutObject 接口编写文档
   ```

2. **设计文档目录结构**

   ```
   /sdk-doc: 设计 SDK 的文档目录结构
   ```

3. **创建文档总索引**

   ```
   /sdk-doc: 创建 API 文档总索引
   ```

4. **审查现有文档**

   ```
   /sdk-doc: 审查并改进桶操作文档
   ```

## Skill 包含的内容

### 1. skill.md - 完整指南

主要包含以下章节：

- **技能用途和适用场景**
- **前置条件**
- **文档结构设计**
- **文档内容规范**
- **文档格式规范**
- **最佳实践**
- **质量检查清单**
- **工作流程**

### 2. templates/ - 文档模板

提供了 3 个标准模板：

- **docs_index_template.md** - 总索引文档模板
- **api_doc_template.md** - API 接口文档模板
- **acceptance_report_template.md** - 验收报告模板

## 文档结构设计

### 推荐目录结构

```
docs/
├── README.md                 # 总索引
├── bucket/                   # 桶操作
├── object/                   # 对象操作
├── multipart/                # 分块上传
├── lifecycle/                # 生命周期
└── encryption/               # 加密
```

### 单个 API 文档包含

1. 方法签名
2. 参数说明（表格）
3. 返回值说明
4. 使用示例（可运行代码）
5. 错误码说明
6. 注意事项

## 文档编写原则

### 1. 以用户为中心

- 从用户视角编写
- 假设用户是第一次使用
- 提供清晰的入门指南

### 2. 保持简洁准确

- 避免冗余信息
- 描述准确清晰
- 突出重点

### 3. 提供完整示例

- 示例代码完整可运行
- 包含错误处理
- 展示常见用法

### 4. 清晰的错误处理

- 列出所有可能的错误码
- 说明错误原因
- 提供解决建议

## 模板使用

### 总索引文档模板

位于 `templates/docs_index_template.md`，包含占位符：

- `{{ SDK_NAME }}` - SDK 名称
- `{{ IMPORT_PATH }}` - 导入路径
- `{{ UPDATE_DATE }}` - 更新日期

使用时替换这些占位符为实际值。

### API 文档模板

位于 `templates/api_doc_template.md`，包含：

- 单个 API 接口的标准格式
- 参数和返回值表格
- 使用示例代码块
- 错误码说明表

### 验收报告模板

位于 `templates/acceptance_report_template.md`，用于：

- 记录文档生成过程
- 统计文档数量和接口数量
- 质量检查结果
- 改进建议
